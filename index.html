Introduction

payutc est un système de paiement avec porte-monnaie électronique avec une architecture client/serveur basée sur des Web Services. Il est utilisé au sein des associations de l'Université de Technologie de Compiègne.

Les utilisateurs peuvent recharger leur porte-monnaie en ligne à l'aide de leur carte bancaire. Ils peuvent ensuite dépenser l'argent présent sur leur compte dans toutes les associations participantes.

Les différents composants du système sont les suivants :

  • server : serveur stockant l'ensemble des comptes, accessible en JSON ou SOAP
  • casper : client de gestion de son compte personnel, avec rechargement, virement et visualisation de l'historique
  • scoobydoo : client d'administration des droits et des produits
  • mozart : client pour les encaissements via une interface tactile
  • pauline : point de vente pour smartphones sous Android

Installation

Chaque application du projet doit être installée séparément, mais une fois le serveur le plus dur est passé une fois que le serveur tourne !

Pour le serveur, la procédure est décrite dans le README du dépôt. Pour toutes les autres applications, la configuration se limite en général au réglage de l'URL du serveur.
